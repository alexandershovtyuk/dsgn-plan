# Finam Design Plans — Design Spec
**Date:** 2026-06-10

## Overview

Статический сайт для объединения OKR-планов дизайн-команд Финам в единый визуальный документ. Гант-чарты по кварталам, иерархия Трек → Goal → Objective → KR. Деплой на GitHub Pages, данные в YAML-файлах в репозитории.

---

## Architecture

### Build pipeline
- `build.js` (Node.js, без фреймворков) читает `data/{год}/teams/*.yaml`
- Парсит YAML (библиотека `js-yaml`), находит файлы через `glob`
- Год берётся из пути файла (`data/2026/...`), не из содержимого
- Все данные сериализуются в JSON и вшиваются в `<script>const DATA = {...}</script>` внутри `dist/index.html`
- Статика копируется в `dist/assets/`

### Client rendering
- `assets/app.js` — ванильный JS, читает `DATA` из глобального скрипта
- Рендерит всё: острова команд, гант-полосы, карточки, фильтры
- Стили — Tailwind CSS (CLI, не CDN): сканирует `src/` и `build.js`, генерирует минимальный `dist/assets/styles.css`
- `tailwind.config.js` расширяет палитру цветами из дизайн-системы при необходимости

### Deployment
- Push в `main` → GitHub Actions запускает `npm ci && npm run build`
- `dist/` публикуется на GitHub Pages через `peaceiris/actions-gh-pages`
- Миграция на GitLab CI тривиальна — тот же `npm run build`, другой деплой-шаг

### Access protection
- Пароль задаётся в `.env` как `SITE_PASSWORD`
- `build.js` читает пароль, хэширует (SHA-256), вшивает хэш в HTML
- При открытии сайта `app.js` показывает экран ввода пароля, проверяет хэш локально
- После верной проверки — сохраняет флаг в `sessionStorage`, показывает сайт

## Data Schema (YAML)

```yaml
team: "Дизайн-система"
owner: "Александр Шовтюк"

display:
  color: "#3b82f6"

tracks:
  - name: "Инфраструктура и архитектура"
    goal: "Дизайн-система переведена на масштабируемую модульную архитектуру"

    objectives:
      - title: "Завершить оптимизацию библиотек"
        description: "Разбить монолитные Figma-библиотеки на модульные"
        quarter: "Q2-Q3"      # Q1 | Q2 | Q3 | Q4 | "Q2-Q3" | null
        key_results:
          - title: "100% компонентов перенесены с сохранением связей"
            description: "Ни один инстанс не теряет связь с библиотекой"
            target: "100%"

kpis:
  - name: "Целостность токенов"
    description: "Доля компонентов без разорванных ссылок на токены"
    target: "100%"
```

**Правила:**
- Статусов нет — ни у objectives, ни у key results
- `quarter: null` или отсутствует → серая полоса «срок не определён»
- `quarter: "Q2-Q3"` → полоса на два квартала
- Год из пути `data/2026/teams/*.yaml`
- Алиасы/аббревиатуры команд не нужны

## UI — Single Page

### Header
- Название «Дизайн-планы · Финам» + год
- `<select>` для выбора команды (опции: «Все команды» + название каждой)
- Тогл «Key Results» (по умолчанию выключен)

### Quarter timeline header
- Полоса кварталов: Q1 / Q2 / Q3 / Q4 — pill-скругление, тёмные оттенки slate
- Полоса месяцев: янв–дек, текущий месяц отмечен красным ▼
- Ширина колонки названия: 240px; остальное — резиновая сетка 12 колонок
- max-width: 1920px, min-width: 1024px

### Team island
Каждая команда — визуально отдельный остров, отделённый от соседних отступом.

**Структура острова (развёрнутого):**

1. **Заголовок команды** — аватар с цветом, название, owner; суммарная полоса на таймлайне (span от первого до последнего квартала команды)
2. **Треки** — для каждого трека:
   - Серая подложка с меткой «Трек N» и Goal-текстом курсивом
   - Тонкая полоса охвата трека на таймлайне
3. **Objectives** — строки под треком:
   - Название + подпись (description краткая)
   - Цветная полоса на нужные кварталы, метка квартала на конце
   - Клик → раскрывается карточка с полным описанием и key results
   - При включённом тогле KR — дополнительные строки под objective с точками-статусами (без статусов, просто список)
4. **KPI-блок** — карточки с name, description, target; визуально отделён от гант-части

**Свёрнутый остров (по умолчанию для всех кроме первой команды или при фильтре):**
- Заголовок + суммарная полоса
- Список Goals (одна строка на трек, курсив)
- Сводка: «Q2–Q4 · 2 трека · 5 objectives»

### Colour coding
- Полоса objectives: `display.color` из YAML команды
- Planned (Q в будущем): тот же цвет, opacity 0.4
- `quarter: null`: серый (#94a3b8)
- Нет разных статус-цветов (active/blocked/completed) — убраны

### Interaction
- Клик по заголовку команды: разворачивает/сворачивает остров
- Клик по строке objective: открывает/закрывает inline-карточку
- Фильтр по команде: скрывает острова других команд, выбранная разворачивается
- Тогл KR: добавляет/убирает KR-строки под objectives

## Scripts

```json
{
  "build": "node build.js",
  "dev": "node build.js --watch"
}
```

`build.js` запускает Tailwind CLI перед финальной сборкой HTML:
```
npx tailwindcss -i src/styles.css -o dist/assets/styles.css --minify
```

`--watch`: пересобирает при изменении `data/**/*.yaml`, поднимает локальный сервер на `localhost:3000`.

---

## Validation

`build.js` мягко валидирует каждый YAML-файл:
- Предупреждение в консоль если отсутствует `team`, `owner`, `display.color`
- Предупреждение если `quarter` содержит неизвестное значение
- Не падает — пропускает невалидный файл с предупреждением

## Out of Scope (v1)

- Авторизация через SSO / OAuth
- Редактор YAML в браузере
- Серверный рендеринг
- CSS/JS фреймворки
- Отдельные страницы на команду
- Статусы objectives и KR
